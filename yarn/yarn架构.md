```
YARN的架构
Client,ResourceManager,NodeManager,ApplicationMaster
master/slave:RM/NM
```
```
Client:向RM提交任务,管理任务等
```
```
ApplicationMaster:每个应用程序对应一个AM，AM向RM申请资源用于在NM上启动对应的Task
```
```
NodeManager:
干活
向RM发送心跳信息，任务执行情况，启动任务
接收来自RM的请求命令
处理来自AM的命令
```
```
ResourceManager:集群中同一时刻对外提供服务的只有一个，负责资源相关。
处理来自客户的请求
启动/监控AM
监控NM
```
```
Container：任务运行的抽象
memory,cpu

```
